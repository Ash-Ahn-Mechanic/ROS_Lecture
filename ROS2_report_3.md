# 2. 로봇과 센서

## 2.1 센서란 무엇인가

센서는 **주변 환경의 정보를 전기 신호로 바꾸는 장치**이다.

예를 들어 사람은 다음과 같은 감각을 가지고 있다.

| 사람 감각 | 로봇 센서 |
|---|---|
눈 | 카메라 센서 |
귀 | 마이크 |
코 | 가스 센서 |
피부 | 온도 / 압력 센서 |

즉

```
환경 → 센서 → 전기 신호
```

이 과정을 통해 로봇은 주변 상황을 알 수 있다.

---

# 3. 로봇의 기본 구조

로봇은 항상 다음 구조로 동작한다.

```
입력 → 처리 → 출력
```

| 단계 | 역할 |
|---|---|
입력 | 센서가 환경 정보를 수집 |
처리 | 컴퓨터가 데이터를 계산 |
출력 | 액츄에이터가 움직임 생성 |

예

```
거리 센서 → 컴퓨터 → 모터
```

---

# 4. PWM 신호

PWM은 **Pulse Width Modulation**의 약자이다.

쉽게 말하면

**전기를 빠르게 켜고 끄면서 값을 조절하는 방법**이다.

예

```
HIGH 5ms
LOW 5ms
전체 10ms
```

주파수 계산

```
1 / 0.01초 = 100Hz
```

PWM은 다음 장치 제어에 사용된다.

- LED 밝기 조절
- 서보 모터 제어
- 모터 속도 조절

---

# 5. 센서 선택 기준

로봇을 만들 때 센서를 선택할 때 다음 요소를 고려한다.

- 가격
- 크기
- 출력 방식 (아날로그 / 디지털)
- 감도
- 정확도
- 응답 속도
- 신뢰성

예를 들어

```
자율주행 자동차 → 매우 정확한 센서 필요
장난감 로봇 → 저렴한 센서 사용
```

---

# 6. 액츄에이터

액츄에이터는 **전기 신호를 받아 실제 움직임을 만드는 장치**이다.

대표적인 예

| 장치 | 역할 |
|---|---|
DC 모터 | 회전 |
서보 모터 | 특정 각도 회전 |
LED | 빛 출력 |

정리

```
센서 = 입력
액츄에이터 = 출력
```

---

# 7. SBC (Single Board Computer)

SBC는 **작은 컴퓨터 보드**이다.

대표적인 예

- Raspberry Pi
- Arduino
- Jetson Nano

특징

- 작은 크기
- 저전력
- 로봇에 많이 사용

---

# 8. SBC와 제어기의 차이

| 구분 | SBC | 제어기 |
|---|---|---|
역할 | 복잡한 계산 | 장치 제어 |
운영체제 | Linux 가능 | RTOS |
성능 | 높음 | 낮음 |

예

```
SBC → 영상 처리
제어기 → 모터 제어
```

---

# 9. IoT (사물인터넷)

IoT는 **인터넷으로 연결된 기기들이 서로 정보를 주고받는 기술**이다.

예

- 스마트 홈
- 스마트 팜
- 스마트 시티

구조

```
센서 → 인터넷 → 서버 → 장치 제어
```

---

# 10. 임베디드 시스템

임베디드는 **기기 안에 들어가는 컴퓨터 프로그램**이다.

예

- 자동차 ECU
- 로봇 제어 프로그램
- 가전제품 제어

특징

- 메모리 제한
- 빠른 처리 필요
- C/C++ 언어 많이 사용

---

# 11. 로봇 좌표계

로봇은 위치 계산을 위해 좌표계를 사용한다.

대표적인 좌표

### 1 Cartesian 좌표

```
(x, y, z)
```

### 2 원통 좌표

```
(r, θ, z)
```

### 3 구면 좌표

```
(r, θ, φ)
```

이 좌표는 로봇 팔 움직임 계산에 사용된다.

---

# 12. ROS (Robot Operating System)

ROS는 **로봇 개발을 위한 소프트웨어 플랫폼**이다.

특징

- 오픈소스
- 모듈 구조
- 로봇 프로그램 개발 편리

ROS 역할

```
센서 ↔ ROS ↔ 액츄에이터
```

즉

**로봇 프로그램끼리 통신을 도와주는 시스템**이다.

---

# 13. ROS2 특징

ROS2는 기존 ROS의 개선 버전이다.

주요 특징

- 실시간 처리
- DDS 기반 통신
- 분산 시스템 지원

---

# 14. ROS2 실습

ROS2 기본 통신 예제

Publisher

```
ros2 run demo_nodes_cpp talker
```

Subscriber

```
ros2 run demo_nodes_cpp listener
```

동작 구조

```
talker → 메시지 발행
listener → 메시지 수신
```

이 구조를

**Publisher / Subscriber 모델**이라고 한다.

---

# 15. 결론

ROS2는 로봇 개발을 위한 매우 중요한 플랫폼이다.

로봇 시스템은 다음 구조로 이루어진다.

```
센서 → 컴퓨터 → 액츄에이터
```

ROS2는 이 장치들이 서로 통신하도록 도와준다.

앞으로 로봇 개발에서는

- 센서
- 임베디드 시스템
- IoT
- ROS2

기술이 매우 중요하게 사용된다.
