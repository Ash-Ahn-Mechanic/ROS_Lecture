# 3. ROS란 무엇인가

ROS는 **Robot Operating System**의 약자이다.

하지만 실제로는 운영체제가 아니라 **로봇 개발을 위한 소프트웨어 플랫폼**이다.

쉽게 말하면

Windows → 컴퓨터 프로그램 개발 도움  
ROS → 로봇 프로그램 개발 도움

이라고 생각하면 된다.

---

# 4. ROS를 쉽게 이해하는 비유

로봇을 사람에 비유하면 다음과 같다.

| 사람 | 로봇 |
|---|---|
| 눈 | 카메라 센서 |
| 귀 | 마이크 |
| 뇌 | 컴퓨터 |
| 근육 | 모터 |

ROS는 이 모든 장치를 연결해주는 시스템이다.

---

# 5. ROS 주요 기능

ROS는 다음 기능을 제공한다.

- 하드웨어 추상화
- 프로세스 간 통신
- 패키지 관리
- 로봇 알고리즘 실행

즉

센서 → 데이터 처리 → 모터 제어

이 과정을 연결한다.

# 6. ROS를 사용하는 이유

## 장점

- 오픈소스
- 거대한 개발 커뮤니티
- 개발 시간 단축
- 다양한 운영체제 지원

지원 OS

- Linux
- Windows
- macOS

## 단점

- 학습 난이도 높음
- 구조가 복잡
- 시스템 리소스 사용량 증가

# 7. ROS 시스템 구조

## Node

ROS 프로그램의 최소 실행 단위

예

- 카메라 노드
- 모터 노드
- 센서 노드

각 노드는 하나의 역할을 수행한다.

---

## Package

노드를 묶어 놓은 단위

예

- 로봇 이동 패키지
- 센서 패키지

---

## Message

노드 사이에서 주고받는 데이터

예

- 속도
- 위치
- 온도

---

## Topic

단방향 데이터 통신

Publisher → Topic → Subscriber

예

카메라 → 영상 데이터 → 화면 노드

---

## Service

요청 / 응답 방식 통신

예

로봇 상태 요청 → 상태 응답

---

## Action

시간이 오래 걸리는 작업 처리

예

로봇 이동

# 8. ROS1 vs ROS2

| 기능 | ROS1 | ROS2 |
|---|---|---|
| 로봇 수 | 단일 로봇 | 여러 로봇 |
| 실시간 제어 | 지원 부족 | 지원 |
| 플랫폼 | Linux | Linux Windows macOS |
| 네트워크 | 안정적 환경 필요 | 불안정 환경 대응 |
| 목적 | 연구 중심 | 산업용 |

# 9. DDS란 무엇인가

DDS = Data Distribution Service

데이터 분산 통신 시스템이다.

쉽게 말하면

여러 컴퓨터가 데이터를 자동으로 공유하는 시스템이다.

---

# 10. DDS 특징

- 산업 표준 기술
- 운영체제 독립
- 언어 독립
- 실시간 통신 지원
- 자동 노드 발견
- 보안 지원

---

# 11. ROS 통신 구조

ROS의 핵심은 메시지 통신이다.

Node → Message → Node

예

센서 노드 → 거리 데이터 → 이동 노드

---

# 12. ROS 시스템 계층 구조

Hardware  
↑  
Operating System  
↑  
DDS Middleware  
↑  
ROS  
↑  
Application

ROS는 OS와 프로그램 사이의 중간 계층이다.

# 13. 독창적인 주장 정리

## 1. ROS는 운영체제가 아니라 SDK

많은 사람들이 ROS를 운영체제로 생각하지만 실제로는 로봇 개발 SDK이다.

근거  
Open Robotics 공식 문서

---

## 2. ROS2에서 DDS 채택이 가장 큰 변화

DDS는

- 실시간 통신
- 산업용 로봇 지원
- 보안 강화

를 가능하게 한다.

---

## 3. ROS는 로봇 개발 시간을 줄인다

기존 개발

통신 / 센서 / 디버깅 모두 직접 구현

ROS 사용

이미 만들어진 도구 활용

# 14. 결론

ROS2는 현대 로봇 개발의 핵심 플랫폼이다.

핵심 특징

- 노드 기반 구조
- 메시지 통신 시스템
- DDS 기반 산업용 통신
- 멀티 플랫폼 지원
